## Appendix A: Payment Gateway Integration — Technical Steps

The platform's escrow engine and wallet ledger are production-ready. When sandbox credentials are obtained, the payment gateway plugs into the existing financial flow with no rebuild required.

### Recommended Saudi Payment Gateways

#### Option 1: HyperPay (Recommended for Production)

Licensed by SAMA, supports mada, Visa, Mastercard, Apple Pay, STC Pay. Most widely used gateway in Saudi Arabia.

**Steps to Get Sandbox Access:**
1. Visit https://www.hyperpay.com → click "Get Started"
2. Submit business registration: company name, CR number, contact person, business type ("Marketplace / B2B Platform"), expected monthly volume
3. HyperPay responds within 1-3 business days with sandbox credentials
4. You receive: Sandbox Entity ID, Sandbox Access Token, Sandbox API endpoint

**Sandbox API Endpoints:**
- Create checkout: `POST https://eu-test.oppwa.com/v1/checkouts`
- Check payment status: `GET https://eu-test.oppwa.com/v1/checkouts/{id}/payment`
- Refund: `POST https://eu-test.oppwa.com/v1/payments/{id}`

**Test Card Numbers (Sandbox):**
- Visa success: 4200000000000000 (any future expiry, CVV: 123)
- Mastercard success: 5200000000000007
- mada success: 4464040000000007
- Declined: 4200000000000018

#### Option 2: Moyasar (Recommended for Pilot — Instant Sandbox)

Saudi-founded, simple REST API, instant self-service sandbox access.

**Steps to Get Sandbox Access:**
1. Visit https://moyasar.com → click "Sign Up"
2. Create account with email — sandbox credentials available immediately
3. You receive: Publishable API Key (frontend), Secret API Key (backend)

**Sandbox API Endpoints:**
- Create payment: `POST https://api.moyasar.com/v1/payments`
- Fetch payment: `GET https://api.moyasar.com/v1/payments/{id}`
- Refund: `POST https://api.moyasar.com/v1/payments/{id}/refund`

**Test Card Numbers (Sandbox):**
- Visa success: 4111111111111111 (any future expiry, CVV: 123)
- mada success: 4464040000000007
- Declined: 4000000000000002

#### Option 3: Tap Payments (GCC-Wide)

**Steps to Get Sandbox Access:**
1. Visit https://www.tap.company → click "Get Started"
2. Create merchant account → navigate to Dashboard > API Keys
3. Copy sandbox Secret Key and Publishable Key

**Sandbox API Endpoints:**
- Create charge: `POST https://api.tap.company/v2/charges`
- Retrieve charge: `GET https://api.tap.company/v2/charges/{id}`
- Refund: `POST https://api.tap.company/v2/refunds`

### PHX Integration Points

The payment gateway connects to the existing codebase at these points:

| Integration Point | Current Flow | Production Flow |
|-------------------|-------------|-----------------|
| **Agent books rooms** | Wallet balance debited → escrow hold created | Payment gateway checkout → agent pays via mada/card → escrow hold created |
| **Pilgrim books via storefront** | Booking created, no payment | Payment gateway checkout → pilgrim pays → booking confirmed |
| **Hotel payout (after check-in)** | Admin triggers wallet credit to hotel | Payout via gateway to hotel bank account (IBAN) |
| **Dispute refund** | Admin resolves → wallet credit to agent | Refund via gateway to original payment method |

**Key files:** `server/routes.ts` (booking endpoint), `server/storage.ts` (wallet transactions)
**New file needed:** `server/services/payment-service.ts` (PaymentService class)
